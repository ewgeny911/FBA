84 Основное предназначение данной программы - это сокращение времени на разработку бизнесс-приложений.
85 В данной программе (далее платформе) реализовано большое количество различных возможностей и функционала, которые обычно требуется
86 для написания большингства бизнесс-приложений. Такие как:
87 Авторизация для подключения к базе данных.
88 Сервер приложений - третье звено при работе с базой данных.
89 Простая рлевая модель. Функционал заведения, редактирвания, удаления пользоватлей и прав на функционал.
90 Простая модель данных. Это аналог фреймворков типа EntityFramework, работающий следующим образом:
91 В базе данных есть три таблицы: fbaEntity, fbaTable, fbaAttribute. В эти таблицы заводятся соотвтетственно сущности, таблицы, и атрибуты сущностей.
92 Это необходимо для того чтобы можно было писать запросы к базе данных в терминах сущносей и атрибутов, а не таблиц и полей.
93 На практике это существенно снижает сложность запросов и увеличивает скорость разработки.
94 Такой запрос называется запросом на языке MSQL, в отличие от обычного SQL.
95 Запросы на языке MSQL гораздо легче писать, редактировоать, вносить изменения.
96 Также значительно снижается количестиво строк (длина запросов).
97 По личным наблюдениям скорость написания запросов на языке MSQL ускоряет написание запросов примерно в 5 раз.
98 Чем более сложный запрос нужно написать, тем больше проявляются преимущества подхода написания запросов в терминах сущностей и атрибутов,
99 по отношению к традиционному написанию запросов на языке SQL. Для простых запросов разницы практически нет.
100 В платформе реализован парсер, который позволяет писать запросы к модели даных на языке MSQL.
101 Есть простой редактор запросов MSQL - упрощенный аналог программ для работы с базами данных и написания запросов,
102 таких как SQL Server Management Studio, PgAdmin, или PL/SQL Developer.
103 
104 Как пример в локальную БД на SQLite заведено несколко таблиц, таких как Document, Contract, ContractAgent
